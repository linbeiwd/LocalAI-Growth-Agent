# Demo Video Script (2:40 target)

The final video must be public on YouTube, include audio, remain under three minutes, and explain both the product and how Codex/GPT-5.6 were used.

## 0:00–0:20 — Problem

**On screen:** Product title, then the pre-filled merchant form.

**Narration:**  
“Small local merchants know their stores, but many do not have the time or team to plan content for Douyin, Xiaohongshu, and WeChat every week. LocalAI Growth Agent turns a short, factual store brief into a practical one-week content workflow.”

## 0:20–0:45 — Input

**On screen:** Slowly highlight store name, university-town location, student audience, real 9.9-yuan offer, goal, and competition fields.

**Narration:**  
“The merchant enters seven facts: the store, business type, location, audience, current offer, goal, and competitive context. This example is a Guizhou barbecue restaurant serving university students.”

## 0:45–1:25 — Generation and results

**On screen:** Click the generate button. Show the audience profile, strategy, Douyin script, Xiaohongshu post, and WeChat copy.

**Narration:**  
“The server validates the input and requests structured JSON. The result starts with an audience profile and strategy, then produces channel-specific content. The prompt prohibits invented prices, rankings, addresses, customer claims, and guaranteed results, so the output stays grounded in the merchant’s brief.”

## 1:25–1:50 — Seven-day plan

**On screen:** Scroll through all seven days and the final advice section.

**Narration:**  
“Instead of stopping at copy, the agent creates one concrete action for each of seven days and ends with promotion advice. This makes the result easier for a busy owner to execute.”

## 1:50–2:25 — Codex and GPT-5.6

**On screen:** Show the GitHub README sections “How Codex and GPT-5.6 contributed,” “Architecture,” and “Validation.”

**Narration:**  
“I built the core project during Build Week with Codex and GPT-5.6. Codex helped inspect the official requirements, plan the product architecture, implement the responsive interface, server route, validation and tests, diagnose MCP, API quota and proxy failures, and verify a real end-to-end request. I made the product decisions: choosing the local-merchant problem, the university-town scenario, the factual-output guardrails, and the available runtime provider. The current default runtime is SiliconFlow DeepSeek-V3.2, with the OpenAI GPT-5.6 Responses route retained as a fallback.”

## 2:25–2:40 — Close

**On screen:** Return to the completed plan and show project name plus repository URL.

**Narration:**  
“LocalAI Growth Agent gives a small merchant a clear starting point for consistent, factual content—without requiring a marketing team. The live demo and source code are available through the submission.”
